① 海外 GitHub Actions runner 实测（权威）

数据由看板仓库内的 probe-mkapi2-latency Action 在 ubuntu-latest（GitHub 海外 runner，Azure 美/欧）上运行写入 latency.json。 在仓库 Actions 页点击 Run workflow 即测一次，推送后刷新本页即可看到结果。这是唯一能真实复现「GitHub Actions 访问国内 API」场景的方式。

点击下方按钮，从你的浏览器直接请求东财 API（no-cors 模式，仅测连通性与耗时，不读取响应体）。 注意：这测的是你浏览器所在网络 → API。若浏览器未挂代理则为「中国→API」（等同 SCF 侧）；若浏览器走 Clash 出海则为「海外→API」的近似。

判读要点

拿到任意 HTTP 响应 = 网络连通、未超时（即使 401/403）。
请求被拒 / 30s 超时 = 才支持「跨洋超时」论断。
若 ① 海外 runner 实测 稳定成功，则原 SCF 迁移理由「跨洋 ReadTimeout」不成立（超时更可能是 API 服务端自身慢/抖动，与平台无关）。 若 ① 出现大量超时，则论断成立、SCF 确有价值。
